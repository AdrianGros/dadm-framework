# DAD-M Education

DAD-M Education is an early public variant of the core DAD-M framework for learning, explanation, analysis, and knowledge-building tasks.

It keeps the same Discover, Apply, Deploy, Monitor cycle, but shifts the emphasis from implementation delivery toward validated learning workflows.

## What stays the same

- the four-phase cycle remains the operating core
- scope and guardrails stay explicit
- outputs should remain reviewable and reusable
- Monitor still decides whether a result is stable enough to keep

## What changes in the education variant

The source material for this variant places extra weight on Discover before explanation. In practice, that means:

- clarify the exact learning question before answering
- define target level and scope before expanding depth
- separate secured knowledge from interpretation and simplification
- keep uncertainty visible instead of smoothing it away
- require some form of application, reconstruction, or transfer before treating a result as stable

## Working principles

The most distinctive principles in the current material are:

- validation before explanation
- discover before depth
- truth before fluency
- concepts before complexity
- application before trust
- monitoring before closure

These principles make the education variant stricter than a normal explanation workflow.

## Typical use cases

- structured topic explanations
- exam preparation with explicit knowledge checks
- learning plans for complex topics
- guided knowledge reconstruction
- teach-back or transfer-oriented study sessions

## Public deliverable shape

The current education material extends the core deliverable format with learning-specific fields.

Typical public structure:

1. Implementation Summary
2. Discover
3. Apply
4. Deploy
5. Monitor
6. Knowledge Status
7. Risks / Open Points
8. Next Steps

## Knowledge-quality markers

The variant introduces explicit markers that help keep the output epistemically clean:

- `Secured`
- `Simplification`
- `Interpretation`
- `Uncertain / context-dependent`

These markers are useful when a learning output needs to show not only what is said, but how stable that knowledge is.

## Why this variant matters

The education variant shows that DAD-M can be adapted beyond implementation-focused work. The same phase logic can support learning tasks, as long as the workflow still protects scope, clarity, evidence, and reviewability.

## Current public status

This variant is documented here in an early public form. The repository currently summarizes the operating logic and deliverable shape rather than publishing a full prompt pack.
