# Quality Principles

These principles summarize the working quality expected from DAD-M artifacts.

## 1. Phase separation

Facts, design, implementation, and validation should stay distinct.

## 2. Artifact-first work

Project knowledge should be externalized into documents, code, prompts, reports, or other reviewable outputs.

## 3. Milestone-based progress

Large work should be split into milestones with explicit scope and acceptance criteria.

## 4. Reproducibility

Outputs should be concrete enough that another person can review what happened and why.

## 5. Human accountability

Humans define goals, constraints, and final decisions. The framework does not replace judgment.

## 6. Conservative documentation

When evidence is incomplete, document limits or use narrower wording instead of stronger claims.
