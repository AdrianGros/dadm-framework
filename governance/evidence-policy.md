# Evidence Policy

This repository is curated conservatively. Public statements should be traceable to available source material and phrased no stronger than the evidence allows.

## Source hierarchy

For this repository, acceptable public statements should be grounded in:

1. the repository content itself
2. the DAD-M starter-kit material used during curation
3. explicit context supplied for the curation task

If sources conflict, prefer the narrower and more defensible wording.

## Allowed usage statement

The usage statement currently maintained in this repository is:

"Used regularly in project context by project managers in a Deutsche Telekom Technik environment."

This is intentionally narrower than any claim of formal approval or standardization.

## Claims that should not appear here

- official adoption claims
- company-wide standard claims
- formal approval claims
- enterprise validation claims
- invented usage metrics
- invented business impact claims

## Publication control

- do not upload raw starter-kit files without checking relevance and sensitivity
- do not publish project-specific documents just because they exist in source material
- if a statement cannot be traced to a source, omit it or mark the limitation clearly
